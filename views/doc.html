Abstract

This specification describes the BUS language, defined as a dictionary of named properties and classes using W3C's RDF technology.
BUS is a project devoted to information about properties of bus and its operational routes using the Web. Regardless of whether information is in people's heads, in physical or digital documents, or in the form of factual data, it can be linked. BUS does not compete with other travel booking and information Web sites; rather it provides a simple approach in which different sites can tell basic parts about bus and its routes.

Status of This Document

The BUS specification is produced as part of the BUS-USU project creation in November-2021. The properties inside will not be changed as this is only a simple project (and possibly only one time use). The BUS RDF namespace URI is fixed and its identifier is not expected to change. But even if the RDF namespace URI is fixed, keep in mind that there is a possibility of us taking down this URI if it is no longer needed.

BUS Vocabulary Terms

This is a complete index of all BUS class and property terms

Class

Class: bus:Bus

Bus - A bus

Status:
stable
Properties:
class facility rFrom rTo price agName agTelp
Description

The class of Bus is a class of busses. This is the base and the only class in BUS namespace. Class Bus is used to represent a commercial bus with its basic information (ex: ticket price, route, facility, etc.) as this class properties.

Properties

Property: bus:class

class - price classification of a Bus

Status:
stable
Domain:
having this property implies being a Bus
Range:
every value of this property is a String
Description

The class property relates a Bus to its price classification.
For example, a Bus class can be "Executive", "Economy", etc.

Property: bus:facility

facility - list of facilities in a Bus

Status:
stable
Domain:
having this property implies being a Bus
Range:
every value of this property is a String
Description

The facility property relates a Bus to its facilities.
Multiple facilities is written in one String separated by a comma ','.
For example, if a Bus facilities includes:

  • AC
  • Toilet
  • Small TV
Then, the value of this property is written as "AC, Toilet, Small TV".

Property: bus:rFrom

rFrom - route from, route starting point of a Bus

Status:
stable
Domain:
having this property implies being a Bus
Range:
every value of this property is a String
Description

The rFrom property relates a Bus to its route starting point.
The route starting point by default is a city name but can be changed as needed.

Property: bus:rTo

rTo - route from, route destination point of a Bus

Status:
stable
Domain:
having this property implies being a Bus
Range:
every value of this property is a String
Description

The rTo property relates a Bus to its route destination point.
The route destination point by default is a city name but can be changed as needed.

Property: bus:price

price - the ticket price for a Bus

Status:
stable
Domain:
having this property implies being a Bus
Range:
every value of this property is a String
Description

The price property relates a Bus to its one-way ticket price.
The value of this property is defined as String for simplicity reasons and flexibility format.

Property: bus:agName

agName - name of a Bus agent

Status:
stable
Domain:
having this property implies being a Bus
Range:
every value of this property is a String
Description

The agName property relates a Bus to the name of agent that sells this bus ticket.
The value of this property is an entity (ex: person, company, etc) name.

Property: bus:agTelp

agTelp - telephone number of a Bus agent

Status:
stable
Domain:
having this property implies being a Bus
Range:
every value of this property is a String
Description

The agTelp property relates a Bus to the phone number of agent that sells this bus ticket.
